MemoryService Architecture Document

1. Introduction

This document describes the architecture for a MemoryService built using C# .NET 9, integrated with a Qdrant vector database running on Docker. The MemoryService will store and retrieve all chat conversations, enabling persistent memory and context awareness.

2. Requirements & Goals

  • Persist every chat conversation.
  • Efficient semantic search and retrieval of past conversations.
  • Integration with Qdrant vector database.
  • Scalable, secure, and performant.

3. High-Level Architecture

The architecture consists of three main components:

  • MemoryService API (.NET 9): Handles requests, manages business logic.
  • Qdrant Vector Database (Docker): Stores vector embeddings of chat conversations.
  • Embedding Generator: Converts chat text into vector embeddings.

4. Detailed Component Breakdown

MemoryService API

Built using .NET 9, exposes REST endpoints for storing and retrieving chat data.

Embedding Generator

Uses a pre-trained embedding model (e.g., OpenAI embeddings or SentenceTransformers) to convert text into vectors.

Qdrant Vector Database

Stores embeddings and metadata, supports efficient similarity search.

5. Data Model & Schema

ChatEntry {
    Id: Guid,
    Timestamp: DateTime,
    ConversationText: string,
    EmbeddingVector: float[]
